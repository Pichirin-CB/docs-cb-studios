# Updating Resources

[Home](/home.md) / Guides / Updating Resources

Use this workflow when replacing an existing CB Studios resource with a newer version.

## Safe Update Checklist

1. Read the product changelog or update notes when available.
2. Back up the current resource folder.
3. Back up edited configuration files.
4. Stop the resource before replacing files.
5. Replace files from the new package.
6. Merge configuration changes carefully.
7. Re-import SQL only if the documentation or update notes require it.
8. Start the resource and test core workflows before opening the server to players.

<div class="docs-callout docs-callout--warning">
  <strong>Do not overwrite custom configuration blindly.</strong>
  <p>Most update problems come from replacing a configured file without merging new options from the updated package.</p>
</div>

## When To Contact Support

Contact support after you can provide:

- Product name and version.
- Framework and server artifact.
- Exact update path.
- Server console errors.
- Client F8 errors when relevant.
- Steps to reproduce the issue.

See [Support](/support.md) for the full support checklist.
