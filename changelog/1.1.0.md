<h2 class="changelog-card-title">Independence, compatibility, and maintainability</h2>
</div>
</div>
<p class="changelog-summary">
This release improves the independence, compatibility, and maintainability of <code>cb_deadzone-extract</code>. The main change is the removal of the <code>hate-bridge</code> dependency, together with an internal refactor and new integration points for inventories and notifications.
</p>
</section>
<div class="changelog-grid">
<section class="changelog-section">
<span class="changelog-section-icon">🔄</span>
<h2 class="changelog-section-name">Updates</h2>
<ul class="changelog-list">
<li>Removed the <code>hate-bridge</code> dependency.</li>
<li>The script is now more independent and easier to install.</li>
<li>Refactored the main internal structure.</li>
<li>Added integration hooks in <code>custom.lua</code> for inventories and notifications.</li>
<li>Added support for custom notification systems through hooks.</li>
<li>Added support for additional inventories: <code>qs_inventory</code>, <code>core_inventory</code>, and <code>tgiann-inventory</code>.</li>
<li>Added new languages: French and Portuguese.</li>
</ul>
</section>
<section class="changelog-section">
<span class="changelog-section-icon">⚡</span>
<h2 class="changelog-section-name">Improvements</h2>
<ul class="changelog-list">
<li>Improved modular design for future expansions.</li>
<li>Improved maintainability for future development.</li>
<li>More flexibility for server developers.</li>
<li>Easier customization without modifying core files.</li>
<li>Better compatibility with different server environments.</li>
</ul>
